### 关键教训 ⚠️

1. **VitePress 配置必须是 .mjs 格式**
   - 不能用 `.ts` 或 `.js`，必须是 `.mjs` (ESM)
   - 否则构建会报错：`"vitepress" resolved to an ESM file. ESM file cannot be loaded by require`

2. **GitHub Actions workflow 路径配置**
   - 仓库本身就是 blog，不需要 `./blog` 子目录前缀
   - 错误示例：`working-directory: ./blog`
   - 正确示例：`working-directory: .` 或直接用 `run: npm ci`

3. **部署验证流程**
   - Actions 成功 ≠ 页面可访问
   - 必须实际 curl 访问验证 HTTP 状态码
   - GitHub Pages 有 CDN 缓存，可能需要 1-2 分钟生效

4. **GitHub Pages 两种模式**
   - Legacy 模式：从指定分支读取（需要手动在 UI 设置）
   - Workflow 模式：使用 actions/deploy-pages（推荐）
   - API 无法切换模式，需要手动在 GitHub UI 操作
