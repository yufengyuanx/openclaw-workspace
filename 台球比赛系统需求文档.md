### 角色详细职责

#### 1. 平台管理员 (Platform Admin)

**职责**：
- 平台整体运营与维护
- 主办方资质审核
- 争议最终仲裁
- 平台数据统计与分析

**可操作**：
| 功能 | 说明 |
|------|------|
| 用户管理 | 封禁/解封违规用户（主办方/选手） |
| 比赛审核 | 审核新注册主办方发布的首个比赛 |
| 争议仲裁 | 处理主办方无法解决的争议 |
| 数据导出 | 导出平台整体运营数据 |
| 系统配置 | 平台级参数配置 |

**可查询**：
- 所有比赛列表及详情
- 所有用户信息
- 平台财务总览
- 运营统计数据

---

#### 2. 主办方 (Organizer)

**职责**：
- 创建并管理比赛
- 选手管理与审核
- 比赛执行与监督
- 奖金发放

**可操作**：
| 功能 | 说明 |
|------|------|
| 注册/认证 | 微信登录，提交主办方资质 |
| 创建比赛 | 填写比赛信息、设置规则 |
| 报名管理 | 审核报名、添加/删除选手 |
| 签到管理 | 开启/关闭签到、手动签到 |
| 抽签编排 | 执行抽签、手动调整对阵 |
| 赛程管理 | 分配球台、调度比赛 |
| 比分录入 | 录入/修改比赛结果 |
| 奖金管理 | 设置奖金、发放记录 |
| 财务管理 | 查看收支、申请提现 |

**可查询**：
- 自己创建的所有比赛
- 参赛选手信息（仅限自己比赛）
- 自己比赛的统计数据
- 个人财务明细

**限制**：
- 只能管理自己创建的比赛
- 删除比赛需确认（赛前 24 小时禁止删除）
- 修改比赛结果需记录原因

---

#### 3. 裁判 (Referee)

**职责**：
- 现场执裁
- 比分录入与确认
- 争议初步处理

**可操作**：
| 功能 | 说明 |
|------|------|
| 比赛确认 | 确认选手到场情况 |
| 比分录入 | 录入比赛结果 |
| 球台释放 | 比赛结束后释放球台 |
| 争议报告 | 提交争议情况给主办方 |

**可查询**：
- 被指派执裁的比赛
- 比赛对阵图
- 个人执裁记录

**限制**：
- 只能操作被指派的比赛
- 不能修改已确认的结果
- 不能管理财务

---

#### 4. 选手 (Player)

**职责**：
- 按时参赛
- 遵守比赛规则
- 确认比赛结果

**可操作**：
| 功能 | 说明 |
|------|------|
| 注册/登录 | 微信一键登录 |
| 报名参赛 | 选择比赛、缴纳报名费 |
| 现场签到 | 扫码签到 |
| 结果确认 | 确认比赛结果（可选） |
| 申诉 | 对结果提出异议 |
| 奖金领取 | 确认接收奖金 |

**可查询**：
- 可报名的比赛列表
- 自己参加的比赛
- 个人战绩与排名
- 个人财务（报名费支出、奖金收入）

**限制**：
- 只能查看公开信息
- 不能修改比赛数据
- 申诉需在比赛结束后 24 小时内

## 📋 项目概述

### 项目名称
台球比赛管理系统（Billiard Tournament System）

### 项目目标
为台球俱乐部、赛事主办方提供一套完整的比赛管理解决方案，实现从报名、签到、抽签、比赛到颁奖的全流程数字化管理。

### 目标用户
- **主办方/裁判**：创建比赛、管理赛程、录入比分、发放奖金
- **参赛选手**：报名、签到、查看对阵、查看比赛结果
- **观众**：查看比赛信息、实时赛况（可选）

## 🏗️ 核心功能模块

### 模块一：比赛创建与配置

#### 1.1 基本信息
- 比赛名称
- 比赛日期/时间
- 比赛地点
- 比赛项目（中式八球、美式九球、斯诺克等）
- 封面图片/海报

#### 1.2 规模设置
| 配置项 | 说明 | 复杂度处理 |
|--------|------|------------|
| 参赛人数上限 | 最大报名人数 | 超员后进入候补名单 |
| 球台数量 | 可用球台数 | 影响赛程编排效率 |
| 裁判数量 | 可用裁判数 | 影响并发比赛数 |

#### 1.3 赛制选择
```
┌─────────────────────────────────────────────────────┐
│ 赛制选择                                            │
├─────────────────────────────────────────────────────┤
│ ○ 单败淘汰赛 (Single Elimination)                   │
│   └─ 输一场即淘汰，适合时间紧凑的比赛               │
│                                                     │
│ ○ 双败淘汰赛 (Double Elimination)                   │
│   └─ 输两场淘汰，有胜者组和败者组                   │
│   └─ 败者组决赛胜者与胜者组冠军决赛                 │
│                                                     │
│ ○ 小组循环 + 淘汰赛                                 │
│   └─ 先分组循环，取前 N 名进入淘汰赛                │
│                                                     │
│ ○ 瑞士轮 (Swiss System)                             │
│   └─ 每轮与战绩相近的选手比赛，适合人数多的情况     │
└─────────────────────────────────────────────────────┘
```

#### 1.4 比赛规则
| 配置项 | 选项 |
|--------|------|
| 抢 N | 抢 1、抢 2、抢 3...抢 N |
| 决赛抢 N | 可与前期不同（如决赛抢 5） |
| 限时 | 每局限时 X 分钟 / 不限时 |
| 让球 | 是否允许让球、让球规则 |

#### 1.5 抽签规则
| 配置项 | 说明 |
|--------|------|
| 种子选手 | 是否设置种子、种子数量 |
| 种子保护 | 种子前几轮不相遇 |
| 同单位回避 | 同一俱乐部选手前期不相遇 |
| 八强重抽 | 进入八强后是否重新抽签 |
| 手动调整 | 是否允许主办方手动调整对阵 |

#### 1.6 名次设置
- 是否设置三四名争夺战
- 录取前 N 名
- 各名次奖金/奖品

```
┌─────────────────────────────────────────────────────┐

│                                                     │
```

```
└─────────────────────────────────────────────────────┘
```

#### 3.2 抽签算法
```python
# 伪代码逻辑
def generate_bracket(players, seeds, rules):
    if rules.seed_protect:
        # 种子选手分散到不同区域
        distribute_seeds(seeds, bracket_size)
    
    if rules.same_club_avoid:
        # 同俱乐部选手前期回避
        avoid_club_clash(players)
    
    if rules.double_elimination:
        # 生成双败对阵表
        return generate_double_elimination(players)
    else:
        # 生成单败对阵表
        return generate_single_elimination(players)
```

#### 3.3 抽签结果确认
- 生成对阵图预览
- 主办方可手动拖拽调整
- 确认后锁定，修改需记录日志
- 推送通知所有选手

#### 3.4 人数不足处理
```
┌─────────────────────────────────────────────────────┐
│ 人数不足处理                                        │
├─────────────────────────────────────────────────────┤
│ 场景 1：报名人数 < 最低开赛人数                     │

│   → 自动通知已报名选手                               │
│                                                     │
```

```
┌─────────────────────────────────────────────────────┐
│ 单场比赛流程                                        │
├─────────────────────────────────────────────────────┤
│ 1. 系统分配球台 → 推送通知双方选手                  │
│                                                     │
│ 2. 选手到场 → 向裁判/主办方确认                     │

│                                                     │
│ 5. 球台释放 → 系统安排下一场比赛                    │
└─────────────────────────────────────────────────────┘
```

| 中途录入 | 比赛中暂停保存 | 长时间比赛 |

### 模块五：双败赛制特殊处理

#### 5.1 胜者组/败者组管理
```
胜者组 Round 1 → 胜者组 Round 2 → 胜者组决赛
       ↓                ↓                ↓
    败者组 R1 ←──  败者组 R2  ←──  败者组决赛
                                       ↓
                              总决赛（胜者组冠军 vs 败者组冠军）
```

#### 5.2 败者组编排
| 规则 | 说明 |
|------|------|
| 败者组轮次 | 胜者组每轮产生败者进入败者组 |
| 败者组比赛 | 败者组内比赛，输者淘汰 |
| 败者组决赛 | 胜者与胜者组决赛败者比赛 |
| 总决赛 | 胜者组冠军 vs 败者组冠军 |
| 双败优势 | 败者组冠军需赢 2 个 BO 才能夺冠（可选） |

#### 5.3 比赛依赖
```
胜者组决赛 完成 ──┬──→ 败者组决赛（等待胜者组败者）
                  │
败者组半决赛 完成 ──┘
                        ↓
                  败者组决赛 完成
                        ↓
                  总决赛（可能需要等待）
```

## ⚠️ 异常情况处理汇总

| 异常场景 | 处理方案 |
|----------|----------|
| 选手迟到 | 自动弃权/等待/手动处理（主办方选择） |
| 人数不足 | 轮空/取消比赛/延期 |
| 比分录入错误 | 修正流程（权限控制 + 日志） |
| 选手中途退出 | 对手晋级，已赛结果保留或作废 |
| 球台故障 | 重新分配球台，比赛暂停/继续 |
| 争议判罚 | 裁判长仲裁，记录争议 |
| 系统故障 | 本地缓存，恢复后同步 |
| 网络中断 | 离线模式，联网后同步 |
