🦞 什么是 OpenClaw？

OpenClaw 是一个开源的 AI 智能体框架，让你能够在本地运行强大的 AI 助手，并连接到各种消息平台（Telegram、Discord、微信等）、工具和服务。

🔧 本地优先

所有数据存储在本地，保护隐私，完全掌控你的 AI 助手。

🌐 多平台支持

连接 Telegram、Discord、WhatsApp、飞书等多个消息平台。

🛠️ 可扩展工具

通过技能和工具系统，轻松扩展 AI 的能力边界。

🤖 多 Agent 协作

支持子 Agent 系统，实现复杂任务的分工协作。

💡 核心理念 OpenClaw 的设计哲学是「Text > Brain」—— 把重要信息写下来，而不是依赖记忆。文件系统就是你的长期记忆。

📦 安装指南

方法一：一键安装脚本（推荐）

# macOS/Linux
curl -fsSL https://openclaw.ai/install.sh | bash

# 验证安装
openclaw --version

方法二：npm 安装

# 全局安装
npm install -g openclaw@latest

# 验证安装
openclaw --version

方法三：Homebrew（macOS）

# 安装 OpenClaw
brew install openclaw

# 验证安装
openclaw --version
✅ 系统要求 Node 22 或更高版本。使用 node --version 检查版本。
✅ 安装后检查 运行 openclaw status 检查系统状态，确保所有组件正常工作。

系统要求

系统 最低要求 推荐配置
macOS 12.0+ 14.0+ (Apple Silicon)
Linux Ubuntu 20.04+ Ubuntu 22.04+
内存 4GB 8GB+
存储 1GB 5GB+

🚀 启动与运行

快速启动（推荐）

# 运行引导向导（自动配置 + 安装服务）
openclaw onboard --install-daemon

# 检查 Gateway 状态
openclaw gateway status

# 打开控制面板
openclaw dashboard

手动启动

# 启动 Gateway 服务
openclaw gateway start

# 查看状态
openclaw gateway status

# 打开控制面板（浏览器访问）
openclaw dashboard
# 或手动访问 http://127.0.0.1:18789/

常用命令

命令 说明
openclaw gateway start 启动 Gateway 守护进程
openclaw gateway stop 停止 Gateway 服务
openclaw gateway restart 重启 Gateway 服务
openclaw status 查看系统状态和使用情况
openclaw configure 交互式配置向导
openclaw skills list 列出所有可用技能
⚠️ 注意事项 Gateway 服务会在后台运行，监听本地端口。确保防火墙设置允许本地连接。

📱 通道系统

通道（Channels）是 OpenClaw 与外界通信的桥梁。你可以配置多个通道，在不同平台与 AI 交互。

支持的通道类型

通道 配置难度 特点
WebChat ⭐ 简单 内置网页，无需配置
Telegram ⭐⭐ 中等 需要 Bot Token
Discord ⭐⭐ 中等 需要 Bot Token + 服务器权限
飞书 ⭐⭐⭐ 较复杂 需要企业应用配置
WhatsApp ⭐⭐⭐ 较复杂 需要 Business API

Telegram 配置示例

# 1. 联系 @BotFather 创建 Bot
# 2. 获取 Bot Token
# 3. 配置 OpenClaw
openclaw configure --section channels

# 选择 Telegram，输入 Bot Token
# 启动后，向你的 Bot 发送 /start
🎯 推荐入门 新手建议从 WebChat 开始，熟悉后再配置 Telegram 或其他通道。

🛠️ 工具系统

工具（Tools）是 OpenClaw 执行具体操作的能力。分为内置工具和外部工具。

核心工具分类

📁 文件系统

read

读取文件内容，支持文本和图片

write

创建或覆盖文件

edit

精确编辑文件内容

apply_patch

跨多文件的结构化补丁（实验性）

💻 运行时

exec

执行 shell 命令，支持后台运行

process

管理后台 exec 会话（list/poll/log/write/kill）

🌐 网络工具

web_search

使用 Perplexity/Brave/Gemini 搜索网络

web_fetch

抓取网页内容（HTML→Markdown）

browser

控制专用浏览器（快照/截图/UI 自动化）

🤖 Agent 协作

sessions_list

列出所有会话

sessions_history

获取会话历史记录

sessions_send

向其他会话发送消息

sessions_spawn

Spawn 子 Agent 执行任务

subagents

管理子 Agent（list/steer/kill）

📱 设备与媒体

nodes

管理配对的移动设备（通知/相机/屏幕）

canvas

控制节点 Canvas（present/eval/snapshot）

image

使用图像模型分析图片

pdf

分析 PDF 文档

📬 消息与自动化

message

发送消息到 Discord/Telegram/WhatsApp 等

cron

管理 Gateway 定时任务

gateway

重启/更新 Gateway 服务

工具组（Tool Groups）

OpenClaw 支持使用 group:* 简写来批量控制工具：

# 可用工具组
group:fs         # read, write, edit, apply_patch
group:runtime    # exec, bash, process
group:sessions   # sessions_list/history/send/spawn/status
group:memory     # memory_search, memory_get
group:web        # web_search, web_fetch
group:ui         # browser, canvas
group:automation # cron, gateway
group:messaging  # message
group:nodes      # nodes
group:openclaw   # 所有内置工具

配置示例

# 只允许文件工具和浏览器
{
  "tools": {
    "allow": ["group:fs", "browser"]
  }
}

# 使用预设配置
{
  "tools": {
    "profile": "coding"     # minimal|coding|messaging|full
  }
}

# 按 Provider 限制工具
{
  "tools": {
    "profile": "coding",
    "byProvider": {
      "google-antigravity": { "profile": "minimal" }
    }
  }
}

工具使用示例

# 读取文件
read path="~/Documents/notes.md"

# 写入文件
write path="~/test.txt" content="Hello World"

# 执行命令（后台运行）
exec command="long-task.sh" yieldMs=10000

# 管理后台进程
process action="poll" sessionId="xxx"

# 搜索网络
web_search query="OpenClaw documentation" count=5

# 浏览器自动化
browser action="open" url="https://example.com"
browser action="snapshot" targetId="xxx"

Loop Detection（循环检测）

防止 Agent 陷入无意义的工具调用循环：

{
  "tools": {
    "loopDetection": {
      "enabled": true,
      "warningThreshold": 10,      // 警告阈值
      "criticalThreshold": 20,     // 严重阈值
      "globalCircuitBreakerThreshold": 30,
      "historySize": 30,
      "detectors": {
        "genericRepeat": true,     // 重复调用检测
        "knownPollNoProgress": true, // 轮询无进展检测
        "pingPong": true           // A/B 交替检测
      }
    }
  }
}

浏览器配置

# 浏览器默认配置
{
  "browser": {
    "enabled": true,
    "defaultProfile": "chrome",  // 默认配置文件
    "profiles": {
      "chrome": {
        "port": 18800,
        "userDataDir": "~/.openclaw/browser/chrome"
      }
    }
  }
}

# 使用 Chrome 扩展（Browser Relay）
# 在浏览器中安装扩展后，使用 profile="chrome"
browser action="snapshot" profile="chrome"
⚠️ 安全提示
  • 使用 exec 时避免破坏性命令（如 rm -rf）
  • 敏感操作使用 security: "allowlist" 限制
  • Loop Detection 默认关闭，建议启用防止无限循环

🤖 子 Agent 系统

子 Agent（Sub-agents）允许你 spawn 独立的 AI 会话来执行特定任务，实现并行处理和任务隔离。

使用场景

🔄 长时间任务

后台运行耗时任务，不阻塞主会话

🔬 专业任务

为特定领域 spawn 专用 Agent（如代码、写作）

🔒 隔离执行

敏感操作在独立会话中运行

⚡ 并行处理

同时执行多个独立任务

基本用法

# Spawn 子 Agent（一次性任务）
sessions_spawn task="分析这个代码库" mode="run"

# Spawn 子 Agent（持久会话）
sessions_spawn task="帮我写一个 Python 项目" mode="session"

# 列出所有会话
sessions_list

# 列出子 Agent
subagents action="list"

# 查看会话历史
sessions_history sessionKey="xxx"

# 向子 Agent 发送消息
sessions_send sessionKey="xxx" message="继续之前的工作"

# 终止子 Agent
subagents action="kill" target="agent-id"

核心参数

# 基本参数
sessions_spawn task="任务描述"
                 label="会话标签"      # 可选，便于识别
                 mode="run|session"    # run=一次性，session=持久
                 runtime="subagent|acp"

# 模型配置
                 model="模型 ID"
                 thinking="on|off"     # 思考模式

# 超时控制
                 timeoutSeconds=3600   # 总超时
                 runTimeoutSeconds=1800 # 运行超时

# 高级选项
                 cwd="/path/to/work"   # 工作目录
                 sandbox="inherit|require"
                 cleanup="delete|keep" # 会话清理策略
                 thread=true           # 线程绑定（ACP）

会话可见性

🔒 安全设置 默认情况下，Agent 可以看到当前会话 + spawn 的子 Agent 会话（tree 模式）。 如果是多用户共享 Agent，设置 tools.sessions.visibility: "self" 防止跨会话浏览。

子 Agent 管理最佳实践

  • 为长期运行的子 Agent 设置合理的 timeoutSeconds
  • 使用 label 参数给子 Agent 命名，便于识别
  • 定期用 subagents list 检查运行中的 Agent
  • 任务完成后及时 kill 释放资源
  • 敏感任务使用 sandbox="require" 隔离
💰 成本注意 每个子 Agent 都会消耗 Token 配额。合理控制并发数量，避免不必要的资源浪费。

常见问题

❓ Gateway 启动失败

检查端口占用：lsof -i :18800，或查看日志 ~/.openclaw/logs/

❓ 技能无法加载

确保 SKILL.md 格式正确，检查 openclaw skills check

❓ 模型连接超时

检查 API Key 配置，确认网络连接，考虑使用本地模型

❓ 子 Agent 无响应

检查是否超时，使用 sessions_list 查看状态
