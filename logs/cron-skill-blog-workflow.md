# 100 Skills 系列博客 - 每日发布工作流

## 10 步执行流程

### Step 1: 确认下一篇编号
- 读取 MEMORY.md 查看当前进度
- 确认下一篇编号和主题
- **验证：** 编号连续，主题明确

### Step 2: 搜索 ClawHub 热门技能
- 搜索 ClawHub 或技能市场
- 确认技能受欢迎程度
- **验证：** 找到 2-3 个相关技能

### Step 3: 读取技能 SKILL.md
- 定位技能文件路径
- 读取核心功能和用法
- **验证：** 理解技能用途和安装方式

### Step 4: 撰写博客草稿
- 按照模板格式撰写
- 包含：概述、核心技能、安全评估、实战案例
- **验证：** 草稿完整，格式正确

### Step 5: 安全审查
- 评估技能权限范围
- 检查代码执行风险
- 评估数据外传可能
- **验证：** 安全等级 ≤ MEDIUM

### Step 6: 最终审核
- 检查技术准确性
- 验证命令和示例
- **验证：** 无错误，可发布

### Step 7: Git Commit & Push
- 添加到 git
- Commit 消息：`[100-skills] #XX: skill-name`
- Push 到远程
- **验证：** push 成功

### Step 8: 更新进度追踪
- 更新 MEMORY.md 进度
- 记录发布日期和安全等级
- **验证：** MEMORY.md 已更新

### Step 9: 部署验证
- 检查博客文件可访问
- 验证格式正确
- **验证：** 文件存在且格式正确

### Step 10: 发送总结
- 生成 10 步执行结果表格
- 发送给用户
- **验证：** 用户收到通知
